Testcheckliste: Rechnungen & Angebote

Umfassende manuelle Testliste für alle Funktionen im Bereich Rechnungen & Angebote.

A) Listenansicht /invoices

Seite lädt mit Statistik-Karten (Offen, Bezahlt, Überfällig)
Filter "Rechnungen" / "Angebote" funktioniert
Suche nach Nummer, Kundenname, Betrag
Status-Badges korrekt (Entwurf, Offen, Bezahlt, Teilbezahlt, Storniert)
Klick auf Rechnung/Angebot öffnet Detailansicht
Archivierte Rechnungen ausblendbar

B) Neue Rechnung erstellen

"Neue Rechnung" Button → navigiert zu /invoices/new
Kundendaten: Kunde aus Dropdown wählen → Adresse, PLZ, Ort, UID werden automatisch ausgefüllt
Belegdatum: Datum setzbar
Leistungsdatum: Sichtbar (nur bei Rechnung, nicht bei Angebot)
Fällig am: Sichtbar (nur bei Rechnung)
Betreff/Referenz: Feld vorhanden und editierbar
Zahlungsbedingungen: Setzbar
Anrede & Titel: Werden vom Kunden übernommen

C) Positionen – Das Kernstück

Die Positions-Bearbeitung hat eine eingebaute Autocomplete-Suche. Ab 2 Zeichen in der Beschreibung erscheinen Vorschläge aus dem Materialkatalog.
Autocomplete: In Beschreibung 2+ Zeichen tippen → Dropdown mit Materialien erscheint
Autocomplete-Auswahl: Material anklicken → Beschreibung, Einheit, Preis, Produktnr. werden automatisch ausgefüllt
"Materialien" Button: Öffnet Dialog mit Materialkatalog
→ Suche funktioniert (Name, Produktnummer, Beschreibung)
→ Kategorie-Filter funktioniert
→ Favoriten (⭐ Sterne) sichtbar und umschaltbar
→ Mehrere Materialien gleichzeitig auswählbar (Checkboxen)
→ Menge pro Material eingebbar
→ "Übernehmen" fügt alle ausgewählten Positionen ein
"Position hinzufügen": Neue leere Zeile wird eingefügt
Menge: Änderbar → Gesamtpreis aktualisiert sich automatisch
Einheit: Dropdown mit allen konfigurierten Einheiten (Stk., m², lfm, Std., ...)
Einzelpreis: Netto-Preis eingebbar → Gesamtpreis berechnet sich
Rabatt %: Pro Position möglich (0–100%)
Langtext: Optionales Textarea für Details unter der Beschreibung
Produktnummer: Wird aus Material übernommen
Reihenfolge: Positionen hoch/runter verschiebbar (↑↓ Pfeile)
Löschen: Position entfernbar (🗑️ Icon)
Gesamtpreis: Automatische Berechnung (Menge × Preis × (1 − Rabatt%))

D) Import-Funktionen

"Aus Angebot": Importiert Positionen aus bestehendem Angebot
"Aus Regiebericht": Importiert Material + Stunden aus Regiebericht
"Aus Projekt": Importiert Materialien aus Projektentnahmen
"Materialien": Öffnet Materialkatalog-Dialog (siehe Sektion C)

E) Summen & Rabatte

Nettobetrag korrekt berechnet (Summe aller Positionsgesamtpreise)
USt. korrekt berechnet (Standard 20%, änderbar auf 0%, 10%, 13%)
Bruttobetrag = Netto + USt.
Gesamtrabatt: Prozent oder Betrag → Abzug korrekt angezeigt
Reverse Charge: Toggle → USt. entfällt, Hinweistext erscheint
Skonto: Wenn konfiguriert, Skonto-Box mit korrekter Berechnung

F) Speichern & Nummernvergabe

Speichern: Rechnung/Angebot wird gespeichert ohne Fehler
Nummernformat: Entspricht der Admin-Konfiguration (z.B. 26001, AN26001)
Status "Entwurf": Dokument kann weiter bearbeitet werden
Nachträgliche Änderungen: Speichern aktualisiert korrekt (kein neues Dokument)

G) Neues Angebot – Unterschiede zur Rechnung

"Neues Angebot" → /invoices/new?typ=angebot
"Gültig bis" Feld sichtbar (statt "Fällig am")
Leistungsdatum NICHT sichtbar
Nummernformat: Mit Prefix AN (z.B. AN26001)
Alle Positions-Features funktionieren identisch wie bei Rechnung

H) PDF-Vorschau & Export

PDF-Vorschau lädt korrekt
Briefkopf: Zeigt konfigurierte Firmendaten (Name, Adresse, Telefon, E-Mail)
Logo: Wird korrekt angezeigt (Position, Größe wie konfiguriert)
Absenderzeile: Korrekt über Empfängeradresse
Empfänger: Kundenanrede, Name, Adresse korrekt
Positionen: Alle Positionen mit Kurztext + Langtext (kursiv) korrekt dargestellt
Summenblock: Netto, USt., Brutto korrekt
Footer: Konfigurierte Fußzeile mit Bankverbindung
Akzentfarbe: Konfigurierte Farbe (Trennlinien, Hervorhebungen)
Seitenumbruch: Bei vielen Positionen korrekt (Footer auf jeder Seite)
UID-Nummer: Wird angezeigt wenn konfiguriert
QR-Code: Wird auf Rechnung angezeigt (EPC-QR für Banking)
PDF-Download: Funktioniert, Datei hat korrekten Namen

I) Sonderfälle & Workflows

Teilzahlung: Bezahlten Betrag eintragen → "Offener Betrag" erscheint im PDF
Storno: Rechnung stornieren → Storno-Grund eingeben → Storno-PDF generieren
Mahnung: Überfällige Rechnung mahnen → Mahnstufe wählen → Mahnungs-PDF
Angebot → Rechnung: Angebot in Rechnung umwandeln (alle Positionen werden übernommen)
Zahlungseingang verbuchen: Teilzahlung oder Vollzahlung eintragen
Mehrere Zahlungen: Mehrere Teilzahlungen eintragbar
Skonto-Box: Bei konfiguriertem Skonto → Anzeige auf PDF mit Berechnung

J) Admin-Konfiguration (wirkt sich auf Rechnungen/Angebote aus)

Diese Einstellungen im Admin-Bereich beeinflussen das Aussehen und Verhalten aller Rechnungen und Angebote.
Rechnungs-Layout → Firmendaten ändern → PDF spiegelt Änderungen wider
Rechnungs-Layout → Logo hochladen → Logo erscheint auf PDF
Rechnungs-Layout → Absenderzeile ändern → Zeigt sich auf PDF
Rechnungs-Layout → Fußzeile (3 Zeilen) konfigurieren → Footer auf PDF
Rechnungs-Layout → Schlusstext Rechnung/Angebot anpassen
Rechnungs-Layout → Akzentfarbe ändern → Farbe auf PDF
Nummernkreise → Prefix/Format ändern → Neue Nummern nutzen neues Format
Bankverbindung → IBAN/BIC ändern → Erscheint auf PDF + QR-Code
Firmen-UID → UID-Nummer setzen → Erscheint auf PDF
Einheiten → Neue Einheit hinzufügen → Erscheint im Einheiten-Dropdown

K) Fehlerfälle testen

Rechnung ohne Positionen speichern → Fehlermeldung oder leere Rechnung?
Rechnung ohne Kunde speichern → Funktioniert (Kundendaten optional)
Sehr viele Positionen (20+) → Seitenumbruch korrekt
Sehr langer Langtext → Wird korrekt umgebrochen
Negative Menge oder Preis → Wird auf 0 begrenzt
Rabatt über 100% → Wird auf 100% begrenzt
Browser-Zurück-Button → Daten gehen nicht verloren
